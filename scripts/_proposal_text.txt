Abstract
Emotion Recognition in Conversations (ERC) is essential for applications such as mental health
monitoring and human-computer interaction. However, real-world deployment faces three simul-
taneous challenges: (1) multimodal conversation data is privacy-sensitive and distributed across
institutions, prohibiting centralized training; (2) emotion annotation is expensive, leaving most data
unlabeled; and (3) modalities may conflict (e.g., sarcasm) or be missing (e.g., sensor failure).
Existing methods address these challenges in isolation — federated learning assumes fully labeled
data, while semi-supervised ERC methods require centralized access. We identify a critical failure
mode: in federated semi-supervised learning, overconfident pseudo-labels generated by softmax-
based methods propagate as “poison” through the global model, causing catastrophic performance
degradation — a phenomenon we term Poison Amplification. We propose Fed-Evidence, the first
framework that replaces the softmax pseudo-labeling paradigm with Evidential Deep Learning
based on Dempster-Shafer theory and Dirichlet distributions. Each modality produces evidence
rather than probabilities, enabling principled epistemic uncertainty quantification. When modalities
conflict, the Dempster-Shafer combination rule produces high uncertainty, causing the system to
refuse pseudo-labeling rather than guess incorrectly. When dissonance is detected, an Evidential
Message Passing GNN borrows certainty-discounted evidence from dialogue history to resolve
ambiguity. We further introduce Epistemic-Aware Federated Aggregation (EAFA), which weights
client contributions by data volume and average epistemic uncertainty, preserving privacy while
defending against poisoned updates. We evaluate on IEMOCAP and MELD, investigating whether
Fed-Evidence can approach centralized supervised performance with only 10% labeled data while
maintaining high pseudo-label purity throughout training.
Keywords: Federated Learning, Semi-Supervised Learning, Evidential Deep Learning, Dempster-
Shafer Theory, Multimodal Emotion Recognition, Conversational AI, Uncertainty Quantification,
Privacy-Preserving AI

1. Introduction
1.1. Literature Review
A. Multimodal Emotion Recognition in Conversations (MERC)
Multimodal ERC has pro-
gressed significantly through graph-based and transformer-based approaches:
• DialogueGCN (Ghosal et al., EMNLP 2019): Models conversational context via Graph
Convolutional Networks with speaker-aware edges. Achieves strong results on IEMOCAP but
requires centralized, fully labeled data.
• MMGCN (Hu et al., ACL 2021): Extends DialogueGCN with multimodal graph construction,
fusing text, audio, and visual modalities at the graph level.
• SDT (Ma et al., AAAI 2024): Sentiment-aware dialogue transformer capturing long-range
dependencies. State-of-the-art on MELD but centralized and fully supervised.
• CORECT (Nguyen et al., AAAI 2023): Cross-modal emotion representations with contrastive
learning.
• SURE (2025): Synergistic Uncertainty-aware REasoning using Mixture-of-Experts for modality-
specific noise handling, but uses variance-based uncertainty rather than principled evidential
methods.
• CMERC (2024): Calibration framework for MERC using curriculum learning and confidence
constraints, but remains centralized and relies on softmax confidence.
Limitation: ALL existing MERC methods require centralized data access and full supervision. None
address the federated or semi-supervised setting.
B. Federated Learning for NLP and Multimodal Tasks
• FedDISC (Qiu et al., NeurIPS 2025): Federated multimodal ERC using diffusion-based
modality recovery. The closest competitor, but assumes 100% labeled data (no SSL). Uses
softmax outputs.
• Pilot (AAAI 2025): Federated fine-tuning of vision-language LLMs. Not for emotion recognition
or conversations.
• FedMultiEmo (2025): Federated multimodal emotion recognition for automotive (facial ex-
pressions + physiological signals). Not conversational, no SSL.
• Fed-MVD (2024): Federated multi-view distillation for medical imaging.
Limitation: Federated multimodal methods universally assume fully labeled data at each client,
which is unrealistic for emotion annotation.
C. Semi-Supervised Federated Learning (FSSL)
• SemiDFL (AAAI 2025): Decentralized federated SSL using neighborhood pseudo-label
consensus + diffusion-based synthetic data generation. Image classification only (CIFAR-10,
STL-10).
• FedPDM (Liao et al., AAAI 2025): Progressive Distribution Matching using Optimal Transport
to align pseudo-label distributions. Image classification only. Provides some OT-based theory.
• (FL)² / SAGE (2024): Federated SSL with label-at-server settings. Unimodal.
Limitation: All FSSL methods are unimodal (image classification). None exploit multi-view agree-
ment for pseudo-label quality. None provide uncertainty-principled pseudo-label gating.
2
D. Evidential Deep Learning and Uncertainty
• Sensoy et al. (NeurIPS 2018): Foundational EDL paper — replacing softmax with Dirichlet
distributions for classification, enabling epistemic uncertainty quantification.
• TMC (Han et al., NeurIPS 2021): Trusted Multi-view Classification using Dempster-Shafer
combination of evidential outputs from multiple views. Centralized, supervised, per-sample
classification.
• TMNR² (Zhang et al., IJCAI 2024): Trusted multi-view learning under noisy supervision using
evidential DNNs. Handles noisy labels but NOT pseudo-labels in SSL. Centralized.
• F-EDL (NeurIPS 2024): Flexible EDL using mixture of Dirichlet distributions for improved
robustness.
Limitation: EDL has been applied to multi-view fusion (TMC) and noisy labels (TMNR²), but NEVER
to federated SSL, and NEVER to conversational emotion recognition.

1.3. Necessity of the Research
The Problem: In federated semi-supervised learning for multimodal conversations, the standard
softmax paradigm creates a vicious cycle: overconfident pseudo-labels →corrupted local models →
poisoned global aggregation →even worse pseudo-labels. We term this Poison Amplification.
3
The Hypothesis: Replacing the softmax probability framework with Evidential Deep Learning
(Dempster-Shafer + Dirichlet) provides a principled “safety valve” — the model quantifies what it
doesn’t know, refusing to pseudo-label ambiguous samples rather than guessing wrong. This breaks
the poison amplification cycle.
Originality: No existing work combines Evidential Deep Learning with Federated SSL for multimodal
conversational emotion recognition. More importantly, the theoretical insight — that epistemic
uncertainty provides a natural defense against federated poison amplification — is entirely novel.
Scientific Significance: If validated, Fed-Evidence would establish that uncertainty-awareness is
not merely a desirable property but a necessary condition for safe federated semi-supervised learning in
multimodal settings. This principle extends beyond emotion recognition to any federated multimodal
SSL application.
2. Research Objectives
Primary Objective
Develop and validate Fed-Evidence, a principled framework for privacy-preserving, semi-supervised
multimodal emotion recognition in conversations that uses evidential uncertainty to ensure pseudo-
label safety in federated settings.
Specific Objectives
1. Theoretical: Provide empirical and analytical evidence that evidential uncertainty gating
(Dirichlet-based) produces fewer pseudo-label errors than softmax confidence gating, particu-
larly under modality conflict conditions. We aim to formalize a DS conflict detection bound
and characterize the Poison Amplification phenomenon.
2. Methodological: Design and implement:
• Evidential Pseudo-Label Gating: Replace softmax confidence thresholding with Dirich-
let epistemic uncertainty gating, incorporating Dempster-Shafer combination for multi-
modal conflict detection.
• EAFA (Epistemic-Aware Federated Aggregation): Volume-and-uncertainty-aware
aggregation that weights clients by data size and average epistemic uncertainty.
• Evidential Message Passing: When DS conflict is high (modality dissonance detected),
leverage a dialogue GNN with certainty-discounted evidence propagation to resolve
ambiguity before pseudo-label decision.
3. Empirical: Evaluate on IEMOCAP and MELD benchmarks to investigate:
• Whether Fed-Evidence can approach centralized supervised performance under low
label regimes (5%–50% labeled data).
• Whether evidential gating maintains higher pseudo-label purity than softmax-based
baselines (e.g., FixMatch) throughout federated training.
• Robustness to sensor failure (missing modalities) and data heterogeneity (non-IID parti-
tioning).
• Note: Specific performance targets will be determined by initial experimental results rather than set a priori.
4
4. Practical Impact: Demonstrate the framework’s applicability to privacy-sensitive domains
where conversation data cannot be centralized (e.g., distributed therapy sessions, multi-site
customer service).

5. Approach and Method
5.1. System Architecture
The architecture is organized into three stages: (I) per-modality evidence computation, (II) evidence
fusion with conflict detection and contextual fallback, (III) uncertainty-gated pseudo-labeling.
Stage I – Per-Modality Evidence (BEFORE fusion): Each modality 𝑚is independently encoded
by a frozen pre-trained model followed by a trainable MLP and a per-modality Evidential Head
(softplus activation). This produces independent evidence e𝑚, belief b𝑚, and uncertainty 𝑢𝑚for
each modality. Critically, evidential heads are placed BEFORE any cross-modal fusion to preserve
modality-specific uncertainty signals.
Stage II – Evidence Fusion: The per-modality belief-uncertainty pairs are combined via Dempster-
Shafer rule, producing fused belief b𝑓, fused uncertainty 𝑢𝑓, and conflict score 𝜅. When 𝜅> 𝜏𝜅
(modality dissonance detected), the system activates the Evidential Message Passing GNN – a
dialogue-context network that borrows evidence from neighboring utterances, weighted by their
own certainty (1 −𝑢𝑗). The borrowed context evidence is then combined with the local evidence
via a second DS combination.
Stage III – Uncertainty-Gated SSL (Dual-Path Gate): Pseudo-labels are accepted via two paths:
(1) Direct acceptance when 𝑢𝑓< 𝜏𝑢AND 𝜅< 𝜏𝜅(modalities agree and are confident); (2)
Contextual acceptance when 𝜅≥𝜏𝜅but the GNN fallback resolves the dissonance (𝑢(final) <
𝜏𝑢,fallback). Samples are rejected only when even dialogue context cannot resolve the ambiguity.
Each client sends model parameters 𝜃𝑘, mean uncertainty
̄𝑢𝑘, and dataset size |𝐷𝑘| to the server.
Server – EAFA: The server aggregates client models using uncertainty-and-volume-aware weights
𝑤𝑘∝|𝐷𝑘| ⋅exp(−𝛽⋅̄𝑢𝑘), then broadcasts the updated global model.

7. Expected Results
Theoretical Contributions
1. Poison Amplification Analysis: Empirical and analytical characterization of how softmax-
based pseudo-labels create error propagation in federated SSL. We aim to show that evidential
gating limits error growth compared to softmax confidence gating, with formal analysis if
tractable.
2. DS Conflict Detection Analysis: Empirical comparison showing that Dempster-Shafer

Methodological Contributions
1. Fed-Evidence Framework: First principled integration of Evidential Deep Learning with
Federated Semi-Supervised Learning for multimodal conversations.
2. EAFA: Volume-and-uncertainty-aware aggregation. Ablation studies will quantify the individual
contributions of the |𝐷𝑘| prior and the exp(−𝛽̄𝑢𝑘) term.
3. Evidential Message Passing: Novel evidence-level GNN with certainty-discounted propaga-
tion. Ablation will compare against standard hidden-state GNN.
Empirical Investigations
1. Label efficiency: Evaluate Fed-Evidence across label ratios (5%–50%) against centralized
supervised baselines (SDT, DialogueGCN) to determine how much labeled data can be saved.
2. Robustness under stress: Test with 10%–50% missing modalities and non-IID severity
sweep (𝛼∈{0.1, 0.5, 1.0}).
3. Pseudo-label purity: Track purity curves over federated rounds, comparing evidential gating
vs. softmax confidence gating.
4. Ablation study: Isolate contributions of each component (EDL heads, DS combination, Yager
fallback, dual-path gate, Evidential MP GNN, ECR loss, EAFA).
5. Discovery: Uncertainty-Heterogeneity interaction — investigate whether epistemic uncer-
tainty naturally correlates with non-IID severity across clients.
Practical Implications
• Demonstrates feasibility of privacy-preserving emotion monitoring across distributed sites.
• Framework is generalizable to any multimodal federated SSL task beyond emotion recognition.
Limitations and Risks
1. Privacy caveat: While EAFA transmits only two scalars ( ̄𝑢𝑘, |𝐷𝑘|), full model parameters
are still communicated, which are vulnerable to gradient inversion attacks. Differential Privacy
(DP-SGD) is an orthogonal defense not explored in this work.
2. Simulation gap: All experiments simulate federated settings on a single GPU. Real distributed
deployment introduces communication latency and system heterogeneity not modeled here.
3. Data sparsity under non-IID: Dialogue-level partitioning with extreme non-IID (𝛼= 0.1)
may result in clients with very few labeled dialogues. If pilot experiments reveal insufficient
signal, we will increase minimum client size or reduce the number of clients.

4. Augmentation design: The proposed weak/strong augmentation strategies for multimodal
dialogue are adapted from unimodal practices. Their effectiveness for conversational ERC
requires empirical validation.
5. Theoretical scope: Formal proofs of convergence bounds for Fed-Evidence require assump-
tions (e.g., Lipschitz smoothness, bounded variance) that may not hold exactly in practice.
We prioritize empirical evidence supplemented by analytical insights where tractable.
